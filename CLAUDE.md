# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Layout

This is a multi-module home video platform. The Go module name is `video-server` even though the repo directory is `ai-video-server`.

- `main.go` + `internal/` + `cmd/` + `migrations/` + `pkg/ffmpeg` вҖ” Go backend (server + worker)
- `admin-web/` вҖ” Vue 3 + Vite admin SPA, served by the Go backend at `/admin` after `npm run build`
- `android-app/` вҖ” phone client (Kotlin + Jetpack Compose)
- `android-tv-app/` вҖ” Android TV client (Kotlin + Jetpack Compose), source-isolated from `android-app`
- `scripts/dev-up.sh` / `scripts/dev-down.sh` вҖ” one-command local stack lifecycle
- `migrations/NNNN_*.{up,down}.sql` вҖ” applied by `dev-up.sh` and tracked in a `schema_migrations` table
- `.run/` вҖ” dev runtime artifacts: `*.pid` files (pid + cmd metadata, tab-separated), `*.log` files, compiled `bin/video-server`
- `docs/`, `CONTEXT.md`, `plan.md`, `AGENTS.md` вҖ” process & knowledge artifacts (see Conventions)

## Backend (Go 1.22)

One binary, two modes selected by `-mode` flag or `APP_MODE` env:

- `server` вҖ” Gin HTTP API on `HTTP_ADDR` (default `:8080`), enqueues background jobs via Asynq
- `worker` вҖ” Asynq consumer that runs transcode + scraping + subtitle tasks; concurrency `MAX_TRANSCODE_WORKERS`

Both modes share the same Postgres pool (`pgx/v5`) and Redis (`go-redis`). `main.go` wires services with two configurer-pattern helpers (`configureAVScraper`, `configureContentTranslation`) so the AV scraper and translation client can be re-tuned without changing constructors.

Run individually (not via `dev-up.sh`):

```bash
go run main.go -mode server
go run main.go -mode worker
go build -o .run/bin/video-server .   # what dev-up.sh produces
```

Tests:

```bash
go test ./...                                 # full suite
go test ./internal/services -run TestScraper  # single package / test
go test -race ./internal/queue
```

OpenAPI/Swagger spec is generated by `cmd/gen-openapi` via `//go:generate` in `main.go`:

```bash
go generate ./...
```

Loadable env file is picked up in priority order: `ENV_FILE` env (overload) вҶ’ `.env` in CWD. Use `.env.example` as the source of truth for required keys.

### Internal package map

- `internal/config` вҖ” env-driven `Config` struct; treat as single source of runtime flags
- `internal/repository` вҖ” `VideoRepository` over pgx; all SQL lives here
- `internal/services` вҖ” domain logic (upload/chunk upload, transcode, scraper + AV scraper framework, translation, recommend, subtitle, image, app, iptv, tv, tv_auth)
- `internal/handlers` вҖ” Gin handlers; `router.go` wires the `API` struct that takes every service as a dependency
- `internal/queue` вҖ” Asynq task definitions, enqueuer, processor (`Register(mux)` maps task types to handlers)
- `internal/middleware`, `internal/response`, `internal/models`, `internal/hashutil`, `internal/utils` вҖ” cross-cutting
- `pkg/ffmpeg` вҖ” ffmpeg wrappers used by the transcode service
- `cmd/cleanup-temp`, `cmd/gen-openapi`, `cmd/import-flick` вҖ” standalone tools

### AV scraper subsystem

`internal/services/scraper_av_*.go` is a multi-site framework (JavDB, JavBus, JavLibrary, ThePornDB and MDCX-derived sites). Per-site URLs, cookies, and ThePornDB token come from `AV_SITE_URL_*`, `AV_SCRAPER_*_COOKIE`, `AV_SCRAPER_THEPORNDB_API_TOKEN`. There is significant test coverage in `scraper_av_*_test.go` and regression fixtures вҖ” when modifying scrapers, prefer extending existing strategy/framework files rather than introducing parallel parsers.

### Translation service

Optional. The Go server expects an OpenAI-compatible endpoint at `TRANSLATION_API_URL` (default `http://127.0.0.1:8000/v1`). `dev-up.sh` will auto-start a local `llama-server` only when the URL points to localhost; otherwise it assumes an external service.

## Frontend (admin-web)

```bash
cd admin-web
npm install
npm run dev      # Vite on 0.0.0.0:5173, proxies /api вҶ’ http://localhost:8080
npm run build    # output in dist/, served by Go at /admin
npm test         # vitest run
```

Override proxy target with `VITE_API_PROXY_TARGET` in `admin-web/.env.development` when the backend is on another host. All API calls go through an axios interceptor that adds `Authorization: Bearer <token>` and expects the `{ code, msg, data }` envelope.

## Android clients

Each client is a self-contained Gradle project. The two trees are intentionally **not** sharing source code.

```bash
cd android-app   && ./gradlew :app:assembleDebug
cd android-tv-app && ./gradlew :tv-app:assembleDebug
```

## One-command dev workflow

```bash
bash scripts/dev-up.sh                       # postgres + redis + migrations + backend + worker + admin dev
bash scripts/dev-up.sh --frontend build      # build admin once and serve via Go /admin
bash scripts/dev-up.sh --frontend off        # skip the SPA
bash scripts/dev-down.sh                     # stop local processes + db/redis containers
ENV_FILE=/abs/path/.env bash scripts/dev-up.sh
```

The script handles: docker image prefetch, postgres readiness wait, idempotent migration apply (via `schema_migrations` rows keyed by file name), backend build into `.run/bin/video-server`, pid-file lifecycle with cmdline verification, and optional translation server boot. Logs land in `.run/{server,worker,frontend,translation}.log`.

## TV App Visual & Focus Language

When touching `android-tv-app/`, consult `CONTEXT.md` for the binding rules. Highlights:

- Focus visuals use a soft blue-cyan glow, never hard/pink strokes. Posters scale on focus; buttons/menus/filters use the shared focus glow.
- Use the shared TV icon-action component for round/small icon buttons вҖ” not Material `IconButton`.
- A focusable node must not stack `tvFocusableGlow()` with an additional `.focusable()` (causes double-focus and "press twice" bugs).
- `FocusRequester.requestFocus()` may only target nodes actually composed at that moment вҖ” don't request the search box focus from home when it isn't mounted.
- TV project's compile graph is intentionally narrow: TV launch/pairing, long video, TV series, `18+`, IPTV, TV settings, and shared infra only. Phone-side home/short video/image-collection/Mine/upload sources may remain on disk as migration reference but stay excluded from the TV build graph.
