# AV 编号规范化策略

## 目标
提升番号识别与候选排序稳定性，减少站点页面写法差异导致的漏抓。

## 基础规则
- 统一大写。
- 统一分隔符（空格/下划线/点 -> `-`）。
- 比较时可使用紧凑 key（去除 `-_. 空格`）。

## 常见变体
- `FC2PPV-123456` / `FC2-PPV-123456` -> `FC2-123456`
- `HEYZO1234` -> `HEYZO-1234`
- `ABP00123` / `ABP-00123` -> `ABP-123`
- `259LUXU-1456` 保持数值前缀+字母组合

## 匹配优先级建议
1. exact_code（完全一致）
2. normalized_code（规范化后一致）
3. title_contains_code（标题包含目标编号）
4. keyword 匹配（标题与关键词模糊匹配）
5. 默认顺序

## 验证建议
- 每次修改编号规则都补表驱动测试。
- 至少覆盖 1 条失败回归样例（曾经识别失败的真实文件名）。
