# TV APK 采用手动分发与发布记录/ABI 双层模型

TV 安装包分发首期采用“管理员上传本地 Release APK + 服务器保存 APK 资产 + 家庭成员登录轻量页手动下载”的方案，不做 TV 客户端内更新，也不做公开下载链接。版本管理以 [[TV 发布记录]] 为上位实体，以 [[TV ABI 安装包]] 为其下具体产物：发布记录承载版本说明、发布时间、发布状态与推荐/回退语义，ABI 包承载 `armeabi-v7a` / `arm64-v8a` 两个真实安装文件。之所以这样选，是因为当前 TV 端明确保留 APK 侧载分发、按 ABI 拆包、家庭成员需显式选包；与此同时，我们又需要保留最近三版回退、已发布/已下线审计、已上传与缺失 ABI 可见性，以及不向非登录用户暴露下载入口。相比直接做应用内更新器、公开下载页或把每个 APK 当独立版本条目，这个双层模型更贴合现有分发现实，也能把“版本语义”“ABI 产物语义”“家庭可见范围”清楚拆开，减少后续实现歧义。
