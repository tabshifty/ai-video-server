# PRD：TV 播放续播提醒（续播提示卡）

- 日期：2026-05-23
- 目标端：`android-tv-app/`（TV App，独立 Android TV 工程，与 `android-app/` 源码隔离）
- 范围：TV 电影 / `18+` / 电视剧播放器（`feature/tv/TvLongFormPlayerScreen.kt`、`feature/tv/TvSeriesPlayerScreen.kt` 及关联纯函数 / token / 单测）

## 1. 用户故事

作为 TV 用户，我在重新进入有播放历史的视频时，希望：

1. **续播被告知**：屏幕左下角出现一个小提示卡，告诉我「上次播放至 X 分 Y 秒」，让我知道当前不是从头开始
2. **可选择从头**：在小卡上看到「从头播放」按钮，按一下就能立刻从 0:00 重新开始
3. **不用强制操作**：默认按钮焦点是「继续观看 (5)」，等 5 秒不动卡片自己消失，视频继续按上次位置播放——不被卡打断
4. **暂停时不被偷跑**：我暂停去接电话时，倒计时不消耗 5 秒；恢复播放后卡片仍在，让我可以再决定
5. **不跨能力打扰**：短视频、IPTV、电视剧自动连播切下一集等场景**不**触发此卡，避免无关上下文的噪声

## 2. 作用域

- **仅** `TvLongFormPlayerScreen`（电影 / `18+`）+ `TvSeriesPlayerScreen`（电视剧）两条已经在做 `watchSeconds → seekTo` 的续播路径
- **仅** 「本次播放第一次实际 seek 到历史位置」且 `initialResumePositionMs ≥ 10_000ms` 时触发
- **不** 改动 `LongFormVideoPlayer` 公共 API 形状
- **不** 改动后端、admin-web、Go 服务、`android-app/`、Android TV 工程外的任何文件

## 3. 非目标（明确不做）

- **N1：短视频 / IPTV 不接入**——它们没有 watchSeconds 续播语义，此卡不适用
- **N2：手机端 `android-app/` 不接入**——TV 工程与手机端源码隔离（AGENTS.md），本期不抽共享层
- **N3：不新增任何全局开关**——5 秒非阻塞卡片不值得占设置项；如未来用户有反馈再加
- **N4：倒计时长度不开放配置**——固定 5 秒，与「连播倒计时长度写死 10 秒不开放设置项」对齐
- **N5：最低门槛秒数不开放配置**——固定 10 秒下限，写在 `TvResumePromptTokens.MinResumeMs`
- **N6：不做「上次选了从头」记忆**——每次进入 player 都是一次新的决策窗口
- **N7：「从头播放」不清空后端历史**——仅 `exoPlayer.seekTo(0L)`，历史的写回继续由播放过程中的常规上报负责
- **N8：不做控制条可见性联动**——bottom padding 固定 156dp，与连播卡现行实现一致
- **N9：不做录屏 / Espresso UI 自动化**——TV 焦点交互复杂、CI 不稳定，本期对齐连播卡的「单测 + 源文 audit + 手测」既有约定
- **N10：不抽公共 `TvPromptActionButton`**——连播卡的按钮副本是 `private`，promote 增加跨文件改动量；本期复制一份 `TvResumePromptActionButton` 私有副本并存，符合「三行类似的代码胜过过早抽象」

## 4. 术语对应（本任务沉淀到 `CONTEXT.md` 的 6 条新术语）

| 术语 | 一句话定义 |
|------|----------|
| `续播提醒` | TV 长视频播放器（电影 / `18+` / 电视剧）在按上次播放位置 seek 历史进度时弹出的提示卡，给用户「从头播放」逃生出口；仅 `initialResumePositionMs ≥ 10_000ms` 时触发；作用域不含短视频与 IPTV。 |
| `续播提示卡` | [[续播提醒]] 的视觉实体：左下角小卡 + 「上次播放至 12:34」主文 + 「继续观看 (5)」「从头播放」双按钮；左侧到屏幕左缘 48dp、底距 156dp、widthIn 320–380dp；与 [[连播提示卡]] 左右镜像、共享 `AppChrome.SurfaceShape` 与 `tvFocusableGlow` 视觉 token。 |
| `续播倒计时` | 续播提示卡可见且可操作时才消耗的 5 秒固定窗口；倒计时数字内嵌在「继续观看 (N)」label，按整秒 `ceil(remainingMs/1000)` 递减。暂停或退出确认提示可见导致卡临时隐藏时，倒计时冻结（不消耗剩余秒数）；恢复播放或退出确认提示消失后若 `remainingMs > 0` 重新显示并接续递减。焦点在续播卡两个按钮之间移动不冻结倒计时，用户未在归零前确认「从头播放」时默认继续观看并永久 dismiss。归零后永久 dismiss 不再回来。 |
| `续播提示卡触发节流` | 同一次 `Composable` 挂载内，每个 `video_id` 第一次实际走 seek-to-history 路径时弹一次，靠复用既有 `resumedFromHistoryVideoId` 状态做节流；同视频 `preservePosition` 路径不触发；电视剧选集面板挑了一个有 watchSeconds 的新集会再次触发；电视剧自动连播 / 手动「下一集」 / `startCurrentEpisodeFromBeginning=true` 路径因 `initialResumePositionMs=0` 自然不触发。本任务不改 `LongFormVideoPlayer` 公共 API。 |
| `续播提示卡永久 dismiss` | 本次播放后续不再渲染续播卡的状态。触发场景：用户点「继续观看」或「从头播放」、倒计时归零、播放错误、电视剧选集面板可见、连播覆盖层 / 连播提示卡可见（防御性互斥）。退出确认提示可见时只临时隐藏，不触发永久 dismiss；若用户未二次 BACK 退出，确认提示消失后仍按剩余倒计时恢复。**不**跨 player 会话持久化（不写 DataStore）。 |
| `续播从头播放语义` | 「从头播放」按钮按下时仅执行 `exoPlayer.seekTo(0L)` + dismiss 卡；**不**重置 `resumedFromHistoryVideoId`、**不**调用 `reportHistory(watchSeconds=0)`、**不**清空后端历史。 |

## 5. 关键决策表（grill-with-docs 结果）

| # | 决策点 | 选项 | 理由 |
|---|--------|------|------|
| Q1 | 作用域 | **电影 + `18+` + 电视剧** | 这三条是 TV 工程内唯一基于 `watchSeconds` 续播的播放器；UnifiedPlayer 短视频 / IPTV 不做位置续播，天然不在范围 |
| Q2 | 触发条件 | **本次真实 seek 到历史位置且 `restorePositionMs ≥ 10_000ms`** | 复用既有 `initialResumePositionMs` 分支；下限 10 秒避免「刚开 3 秒退出」的噪声；无上限即便接近完播也弹 |
| Q3 | 视觉语言 | **连播卡同款视觉（卡片背景/圆角/光感），独立兄弟组件** | 「系统主动告知 + 倒计时」语义同构；避免居中「夜台玻璃面板」语义错位 |
| Q4 | 双按钮 / 单按钮 | **双按钮** | 「从头播放」之外加一个显式「继续观看」出口，与连播卡两按钮形态对称 |
| Q5 | 按钮顺序与焦点 | **左 `继续观看 (5)`（抢焦点 + 倒计时）→ 右 `从头播放`；焦点移动不冻结倒计时** | 与连播卡「左为默认 + 倒计时」严格对称；默认手感是「继续看」，避免 OK 键误触从头；续播卡是非阻塞提示，不是必须决策的确认弹窗 |
| Q6 | 卡片主文 | **`上次播放至 12:34`（按时长 `m:ss` 或 `H:mm:ss`），单行不加标题/百分比** | 时间戳是用户重新进入时唯一缺失的信息；其它内容（标题/封面/百分比）属于 10-foot 噪声 |
| Q7 | 状态守卫 | **暂停 / 退出确认 → 临时隐藏；错误 / 电视剧选集面板 / 连播覆盖层 / 连播卡 → 永久 dismiss；归零 → 永久 dismiss** | 与连播卡「状态守卫」模式对齐，BACK 只是退出意图确认，不代表用户已处理续播选择；字幕 / 音轨夜台面板不进入本任务状态机 |
| Q8 | 多次触发 | **每个 `video_id` 第一次 seek-to-history 弹一次；选集面板选另一集会再次弹；同视频 preserve-position 路径不弹** | 复用 `resumedFromHistoryVideoId` 节流，不引入新 state |
| Q9 | 全局开关 | **不加** | 5 秒非阻塞卡片不值得占设置项 |
| Q10 | 位置 | **`Alignment.BottomStart` + `padding(start = 48.dp, bottom = 156.dp)`，与连播卡左右镜像** | 双卡空间天然互斥；不联动控制条可见性 |
| Q11 | 代码结构 | **`TvResumePromptCard.kt`（UI）+ `TvResumePrompt.kt`（纯函数+token），按钮内联私有副本** | 与 `TvAutoplayPromptCard.kt` / `TvSeriesAutoplay.kt` 拆分对称；不强抽公共按钮 |
| Q12 | CONTEXT.md 术语 | **追加 6 条去实现细节的术语，不改既有** | 续播术语集中在一处，未来可索引 |
| Q13 | 验收 / 版本 | **单测 + 源文 audit + 18 条手测；versionCode 62→63、versionName 0.1.61→0.1.62** | 对齐 `AGENTS.md` 版本递增规则与连播卡既有验收形态 |

## 6. 验收标准（QA scenarios）

### 6.1 电影 / `18+` 单视频路径

- **H1**：进入电影详情页（视频 `watchSeconds = 754` 秒，即 12:34）→ 点播放
  - ✓ 视频按 12:34 续播开始
  - ✓ 左下角续播卡出现：主文「上次播放至 12:34」，按钮行「继续观看 (5)」+「从头播放」
  - ✓ 焦点抢占在「继续观看」上（TV 焦点蓝青光感可见）
  - ✓ 倒计时数字每秒递减：5 → 4 → 3 → 2 → 1
- **H2**：H1 续，5 秒不操作
  - ✓ 倒计时归零，卡 fadeOut 自动消失
  - ✓ 视频继续从约 12:39 播放（不发生第二次 seek）
- **H3**：H1 续，3 秒时按 OK（焦点在「继续观看」上）
  - ✓ 卡立刻消失
  - ✓ 视频继续从当前位置播放
- **H4**：H1 续，DPad RIGHT → OK（在倒计时归零前选「从头播放」）
  - ✓ 卡立刻消失
  - ✓ 视频从 0:00 重新开始播放
  - ✓ 同视频后续不再弹卡（`resumedFromHistoryVideoId` 已置位）
- **H5**：H1 续，3 秒时按遥控器 PAUSE
  - ✓ 卡临时隐藏
  - ✓ 倒计时冻结在剩余 2 秒
- **H6**：H5 续，按 PLAY
  - ✓ 卡再次出现，倒计时从 2 继续递减
- **H7**：H1 但目标是 `18+` 视频
  - ✓ 与 H1 行为完全一致

### 6.2 电视剧路径

- **H8**：进入电视剧详情页（当前集 `watchSeconds = 320` 秒，即 5:20）→ 点播放
  - ✓ 当前集按 5:20 续播开始
  - ✓ 左下角续播卡出现：主文「上次播放至 5:20」
- **H9**：H8 续，让视频自然播到剩余 10 秒（T-10）
  - ✓ 续播卡早已 dismiss（5 秒后自动）
  - ✓ 右下角连播卡正常出现，**不与续播卡同屏**
- **H10**：电视剧播放器内打开选集面板，选另一集（该集 `watchSeconds = 522` 秒，即 8:42）
  - ✓ 切到新集 + 续播卡再次出现，主文「上次播放至 8:42」
- **H11**：电视剧控制条点「下一集」按钮
  - ✓ 切到下一集
  - ✓ **不**出现续播卡（`startFromBeginning=true` → `initialResumePositionMs=0`）
- **H12**：电视剧自动连播切到下一集（T-10 倒计时归零）
  - ✓ 切到下一集
  - ✓ **不**出现续播卡

### 6.3 边界与互斥

- **H13**：进入有 `watchSeconds = 6` 秒（低于 10 秒下限）的电影
  - ✓ 视频按 6 秒续播
  - ✓ **不**出现续播卡
- **H14**：进入有 `watchSeconds = 0` 的新电影
  - ✓ 视频从 0:00 开始
  - ✓ **不**出现续播卡
- **H15**：电影续播卡显示中按 BACK
  - ✓ 续播卡永久 dismiss（不再回来）
  - ✓ 「TV 播放器退出确认」第一次提示出现
  - ✓ 用户撤销退出确认（5 秒内不再按 BACK 自然消失，或方向键唤起控制条）后，续播卡**不**再次显示——视为「用户进入了别的决策流」永久退出
  - ✓ 再次 BACK 退出 player（BACK 不被本卡劫持）
- **H16**：电影续播卡显示中打开字幕 / 音轨夜台玻璃面板
  - ✓ 续播卡永久 dismiss
  - ✓ 字幕 / 音轨面板正常显示
  - ✓ 关闭夜台玻璃面板后续播卡**不**回来（与连播覆盖层、选集面板同一守卫语义）
- **H16b**：电视剧续播卡显示中打开选集面板
  - ✓ 续播卡永久 dismiss
  - ✓ 选集面板正常显示
- **H17**：同一视频续播卡 dismiss 后，关闭浮层（退出确认 / 字幕音轨 / 选集面板任一）回到播放器
  - ✓ 续播卡**不**再次出现（同视频本次会话已永久 dismiss）
- **H18**：退出 player 回详情页 → 再次点播放（同一视频）
  - ✓ 续播卡再次出现（重新挂载 → 重新有机会）
